### 🧪 ASCII Format Analysis
After thorough analysis of Dean's logs and Java source code, confirmed:
- **ASCII format is real** - comes from HomGar cloud backend
- **Multiple firmware versions** use different payload formats
- **No Java parsing found** - format generated server-side
- **Raw API response** shows ASCII format: `1,-84,1;0,149,0,0,0,0|0,6,0,0,0,0`

### 📋 Payload Formats Supported

#### **HTV213FRF/HTV245FRF Valve Devices**
- **Hex**: `11#17E1CE0019D8001AD8001D201E2021B700000000...`
- **ASCII**: `1,-84,1;0,149,0,0,0,0|0,6,0,0,0,0`

#### **HCS021FRF Soil Sensors**
- **Hex**: `10#E1A200DC0185AB02881FC6600600FF0FFA28F718`
- **ASCII**: `1,-73,1;694,70,G=292478`
